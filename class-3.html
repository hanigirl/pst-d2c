Skill = a set of repeated instructions

When?

When do you create a skill?

Proactively

When a task keeps repeating

  • Writing copy for X
  • Parsing invoice emails
  • Comparing prices
  • Analyzing stocks
  • Writing a sales page
Retroactively

When an output worked well and you want to recreate it

  • You dialed the AI in to design a banner — and want to lock the visual language
  • You asked it to analyze data and loved the structure Claude produced
How?

How do you create a skill?

You just ask Claude

Claude, I want to create a skill that does [some sisyphean repeated task]. Here are the instructions.
I created this document.
Change this here, drop that.
Fixed it — take a look.
Checking…
Save it?

Skills for the design-to-code flow

What skills to use?

Two flavors — they do very different things

Context skill

How to think

Loaded as context before the AI acts. The AI does the work — the skill just shapes how it thinks.

  • Scan the codebase first
  • Read X before doing Y
  • Gather context from a file
  • Propose changes based on best practices
  • Apply brand or project rules
remotion-best-practices brand-voice project-rules
Action skill

What to do

Wraps a tool or API. The skill actually executes something — Figma canvas, files, shell, MCP server.

  • Write directly to the Figma canvas
  • Generate components from a JSON spec
  • Sync tokens between code & Figma
  • Edit a design programmatically
  • Run multiple agents in parallel
figma-use figma-generate-design cc-figma-component
