Definition

Agent = a task that finishes itself

A multi-step job that runs end-to-end — using its own toolkit and reaching for resources when it needs them.

Has its own Skills Pulls Resources on demand Knows when it's done
Composition

Main task → sub-agents

Main agent
Ship the weekly newsletter
Sub-agent · 01
Read the brand voice
Pull tone, do/don'ts, and past examples from the knowledge base.
Sub-agent · 02
Draft the issue
Use the newsletter-draft skill on this week's raw material.
Sub-agent · 03
Generate subject lines
Run the subject-line skill and rank the candidates.
Each agent becomes a sub-agent. The success of one is the success of the whole.
Build

How to build an agent

Four things you tell Claude when you spec it out

1
Knowledge

What it should know

Define the agent's domain expertise and the experience it draws on — what counts as the "right" answer here.

2
Action

What it should do

Spell out the process step by step — the method, the tools (skills), and the order of operations.

3
Cadence

When and at what pace

Set the rhythm — once a week, on a trigger, or always-on — and how aggressively to push toward completion.

4
Decisions

How to react to each case

Define the edge cases up front: if X, then Y — not X, but Y.
